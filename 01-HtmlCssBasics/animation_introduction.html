CSS-Animationen

Hier ein paar Beispiele für Animationen mit CSS.

Jedes Beispiel zeigt links die CSS-Formatierung und rechts die Animation, die per Button gestartet werden kann.

Das Demonstrationsobjekt in den folgenden Beispielen ist immer ein <div>-Tag, aber es kann grundsätzlich auch jeder andere Tag animiert werden.

Die Objekte haben eine CSS-Klasse mit einem Basis-Styling zugewiesen bekommen (ergibt einen grünen Kreis).

.gd-demo {
    display: block;
    width: 4em;
    height: 4em;
    background-color: #6d9;
    border-radius: 50%;
}

Jedes Objekt hat eine eigene CSS-Kasse, z.B. "gd-demo-1", erhalten.

Durch Betätigung des "Start"-Buttons im jeweiligen Beispiel wird dem Objekt vorübergehend die CSS-Klasse "gd-active" hinzugefügt.

Die jeweilige Animation ist für die Kombination der objekteigenen CSS-Klasse und "gd-active" hinterlegt, z.B. "gd-demo-1 gd-active".

Einfacher Farbwechsel

Animationskonfiguration

Die CSS-Konfiguration für eine Animation steuert die Dauer und das Timing der Übergänge zwischen einzelnen "Schlüsselszenen" (@keyframes).

Wenn nicht anders angegeben, ist ease das Standard-Timing (beschleunigen zur Mitte und wieder verlangsamen).

Für eine Animation müssen mindestens die Dauer (animation-duration) und der Name der Schlüsselszenendefinition (animation-name) angegeben sein.

.gd-demo-1.gd-active {
    animation-duration: 3s;
    animation-name: gd-keyframe-demo-1;
}

Key Frames

"Schlüsselszenen" definieren die wesentlichen Zustände, die eine Animation durchlaufen soll.

Es muss mindestens zwei Schlüsselszenen geben: 0% und 100%, also Anfangs- und Endzustand.
0% entspricht dem Zustand bei 0s, 100% in diesem Fall dem Stand nach 3s.

Weitere Zustände und wann diese erreicht sein sollen, werden mit %-Angaben zwischen 0 und 100 definiert.
Der Zustand bei 50% wäre bei dieser Animation nach 1,5s erreicht.

@keyframes gd-keyframe-demo-1 {
    0% {background-color: #6d9;}
    25% {background-color: #d96;}
    50% {background-color: #96d;}
    75% {background-color: #9d6;}
    100% {background-color: #6d9;}
}

Einfacher Farbwechsel - unendlich wiederholt

Animationskonfiguration

Man kann definieren, wie oft sich eine Animation wiederholen soll (animation-iteration-count).
standardmäßig wird eine Animation nur einmal abgespielt.
Das Objekt kehrt nach der Animation in den Anfangszustand zurück (0% key frame).

.gd-demo-2.gd-active {
    animation-duration: 3s;
    animation-iteration-count: infinite;
    animation-name: gd-keyframe-demo-2;
}

Key Frames

@keyframes gd-keyframe-demo-2 {
    0% {background-color: #6d9;}
    25% {background-color: #d96;}
    50% {background-color: #96d;}
    75% {background-color: #9d6;}
    100% {background-color: #6d9;}
}

Einfache Bewegung

Objektkonfiguration

Um die Position verändern zu können, ist die Eigenschaft position notwendig.

.gd-demo-3 {
    position: relative;
}

Animationskonfiguration

Wie beim Farbwechsel ist hier keine bestimmte Timing-Funktion angegeben.
Deshalb wird auch hier ease verwendet.

ease führt zu Positionswechseln in "weichen" Schritten mit Übergängen.

.gd-demo-3.gd-active {
    animation-duration: 3s;
    animation-name: gd-keyframe-demo-3;
}

Key Frames

@keyframes gd-keyframe-demo-3 {
    0% {left: 0em;}
    25% {left: 2em;}
    50% {left: 4em;}
    75% {left: 6em;}
    100% {left: 8em;}
}

Einfache Bewegung - linear

Objektkonfiguration

.gd-demo-4 {
    position: relative;
}

Animationskonfiguration

Um die Bewegung flüssig erscheinen zu lassen, wird die Timing-Funktion linear benötigt.

.gd-demo-4.gd-active {
    animation-duration: 3s;
    animation-timing-function: linear;
    animation-name: gd-keyframe-demo-4;
}

Key Frames

@keyframes gd-keyframe-demo-4 {
    0% {left: 0em;}
    25% {left: 2em;}
    50% {left: 4em;}
    75% {left: 6em;}
    100% {left: 8em;}
}

Sprite Sheet Transitioning

Objektkonfiguration

Für Übergangsanimationen bei Sprite Sheets benötigt das Objekt ein Hintergrundbild.

Die Größe des Objekts muss genau ein Teilbild abdecken.
Die Animation schiebt dann das Hintergrundbild schrittweise so, dass immer ein anderes Teilbild zu sehen ist.

.gd-demo-5 {
    width: 256px;
    height: 256px;
    background-image: url(./images/fox_run_90.png);
}

Animationskonfiguration

Um echte Bewegung mit Teilbildern zu simulieren, müssen die Teilbilder in Schritten ausgetauscht werden.

ease und linear würden den Schiebevorgang des Hintergrundbilds mit animieren. Das ist hier nicht hilfreich.

Deshalb verwendet diese Animation die steps-Funktion, um wie bei einem Spielfilm die Einzelbilder auszutauschen.

.gd-demo-5.gd-active {
    animation-duration: 4s;
    animation-timing-function: steps(1, jump-start);
    animation-iteration-count: infinite;
    animation-name: gd-keyframe-demo-5;
}

Key Frames

Hier werden die Teilbilder im Hintergrund des Objekts verschoben, indem die Position des Hintergrundbildes im Anzeigeobjekt geändert wird (background-position: x y).

Wir haben hier 16 Teilbilder (Frames), die in 4s gewechselt werden
=> 4 Frames per Second (FPS).

@keyframes gd-keyframe-demo-5 {
    0% { background-position: 0px 0px; }
    6.25% { background-position: -256px 0px; }
    12.5% { background-position: -512px 0px; }
    18.75% { background-position: -768px 0px; }
    25% { background-position: 0px -256px; }
    31.25% { background-position: -256px -256px; }
    37.5% { background-position: -512px -256px; }
    43.75% { background-position: -768px -256px; }
    50% { background-position: 0px -512px; }
    56.25% { background-position: -256px -512px; }
    62.5% { background-position: -512px -512px; }
    68.75% { background-position: -768px -512px; }
    75% { background-position: 0px -768px; }
    81.25% { background-position: -256px -768px; }
    87.5% { background-position: -512px -768px; }
    93.75% { background-position: -768px -768px; }
    100% { background-position: 0px 0px; }
}
